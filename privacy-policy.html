Privacy Policy

MusicBee Remote application privacy policy

MusicBee Remote does not collect or store, or send any personal information on it's users.

The MusicBee plugin contacts the kelsos.net domain when the user checks for updates, there is no data collection during the call, it is purely used as means to determine the latest version available.

From v1.0.0 the application also includes the Crashlytics SDK.

For any concerned user, there is also a version build without the Crashlytics SDK available on the GitHub releases page of the project.

Requested Permissions

  1. Read phone status and identity (PHONE STATE permission), is only used to determine an incoming call as part of the feature to perform action on an incoming call. It can be safely disabled without affecting anything other than the (Pause/Stop/Reduce Volume) on call functionality. The functionality can be found on the following line: RemoteBroadcastReceiver for more information
  2. Full network access, view network connections and view wifi connections are required in order to implement the remote functionality as it is based on an active network connection and without it it cannot function.
  3. Allow Wi-Fi Multicast reception, on the newer version is used as a means of automatic service discovery, (scanning functionality)
